# 验收指南

本文档提供完整的功能验收步骤，用于验证部署是否成功。

## 快速验收

### 方式 1：Smoke Test（推荐，最快）

运行最小回归测试（约 5 秒）：

```bash
npm run smoke
# 或
node scripts/smoke-test.mjs

# 指定 URL
npm run smoke -- http://your-server:3456
```

检查项（11 项）：
- Health endpoint
- 首页重定向
- 访客/后台登录页可访问
- 登录 API 参数校验
- 后台 API 鉴权
- 站点信息/按钮/背景 API

### 方式 2：Acceptance Test（完整）

运行完整验收测试：

```bash
# 标准模式（9 项检查）
npm run acceptance

# 快速模式（只检查关键链路）
npm run acceptance -- --smoke

# 完整模式（13 项检查，包括更多 API 测试）
npm run acceptance -- --full

# 指定 URL
npm run acceptance -- http://your-server:3456
```

### 方式 3：Preflight Check（发布前检查）

执行发布前完整检查流程：

```bash
# 完整模式（包含 build、acceptance、运维脚本检查）
npm run preflight

# 快速模式（跳过构建，只检查运行态）
npm run preflight:quick

# 指定 URL
npm run preflight -- http://your-server:3456
```

检查项：
1. Build check（quick 模式跳过）
2. Health check
3. Smoke test
4. Acceptance test（quick 模式跳过）
5. Backup script check
6. Maintenance scripts check
7. Environment check
8. Storage directories check

如果全部通过，继续手动验收关键流程。

## 详细验收步骤

### 1. 基础访问验收

#### 1.1 首页访问
- [ ] 访问 `http://your-server:3456/`
- [ ] 未登录状态下自动重定向到 `/access`
- [ ] 页面正常加载，无 500 错误

#### 1.2 健康检查
- [ ] 访问 `http://your-server:3456/api/health`
- [ ] 返回 JSON: `{"status":"ok","database":"connected"}`
- [ ] HTTP 状态码 200

#### 1.3 站点信息 API
- [ ] 访问 `http://your-server:3456/api/site`
- [ ] 返回包含 `title`, `subtitle`, `description` 的 JSON

---

### 2. 访客登录验收

#### 2.1 登录页面
- [ ] 访问 `/access`
- [ ] 页面显示密码输入框
- [ ] 页面样式正常（无错位）

#### 2.2 正确密码登录
- [ ] 输入访客密码（默认: visitor123）
- [ ] 点击登录
- [ ] 成功跳转到首页 `/`
- [ ] 首页显示站点标题、副标题

#### 2.3 错误密码处理
- [ ] 输入错误密码
- [ ] 显示错误提示"密码错误"
- [ ] 页面不跳转

#### 2.4 登录失败限制
- [ ] 连续输入 5 次错误密码
- [ ] 第 6 次提示"密码错误次数过多，请 15 分钟后再试"
- [ ] 15 分钟内无法继续尝试

#### 2.5 会话保持
- [ ] 成功登录后关闭浏览器
- [ ] 重新打开首页
- [ ] 仍然保持登录状态（7 天内）

#### 2.6 登出功能
- [ ] 首页如有登出按钮，点击后登出
- [ ] 或清除 cookie 后需重新登录

---

### 3. 管理员登录验收

#### 3.1 登录页面
- [ ] 访问 `/admin/login`
- [ ] 页面显示密码输入框
- [ ] 样式正常

#### 3.2 正确密码登录
- [ ] 输入管理员密码（默认: admin123）
- [ ] 成功跳转到后台 `/admin`
- [ ] 显示后台管理界面

#### 3.3 错误密码处理
- [ ] 输入错误密码
- [ ] 显示错误提示
- [ ] 同样适用 5 次失败限制

#### 3.4 会话隔离
- [ ] 访客已登录状态下访问 `/admin`
- [ ] 仍需输入管理员密码（会话不互通）

#### 3.5 登出功能
- [ ] 点击后台"退出登录"
- [ ] 成功登出，跳转到登录页
- [ ] 再次访问 `/admin` 需重新登录
