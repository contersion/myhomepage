# 上线检查清单

在将 My Homepage 部署到生产环境前，请逐项确认以下检查项。

## 环境配置检查

### Session Secrets
- [ ] `VISITOR_SESSION_SECRET` 已修改，不是默认值
- [ ] `ADMIN_SESSION_SECRET` 已修改，不是默认值
- [ ] 两个 Secret 长度都 >= 32 位
- [ ] 两个 Secret 不相同
- [ ] Secret 使用随机字符串（如 `openssl rand -base64 48`）

### 初始密码
- [ ] `VISITOR_PASSWORD` 已修改，不是 `visitor123`
- [ ] `ADMIN_PASSWORD` 已修改，不是 `admin123`
- [ ] 密码长度 >= 6 位
- [ ] 密码不包含常见弱口令（如 123456, password 等）

### 数据库配置
- [ ] `DATABASE_URL` 路径正确
- [ ] 数据库目录已创建（`storage/data/`）
- [ ] 数据库目录有正确权限

### 上传配置
- [ ] `UPLOAD_DIR` 路径正确
- [ ] 上传目录已创建（`storage/uploads/`）
- [ ] 上传目录有正确权限
- [ ] `MAX_UPLOAD_SIZE` 设置合理（默认 5MB）

---

## Docker 配置检查

### Volume 挂载
- [ ] `./storage/data:/app/storage/data` 已挂载
- [ ] `./storage/uploads:/app/storage/uploads` 已挂载
- [ ] 宿主机目录已创建

### 端口映射
- [ ] 外部端口配置正确（默认 3456）
- [ ] 端口未被其他服务占用
- [ ] 防火墙已开放该端口（如需要）

### 健康检查
- [ ] Dockerfile 包含 HEALTHCHECK
- [ ] docker-compose.yml 包含 healthcheck 配置
- [ ] 健康检查端点 `/api/health` 可访问

### 重启策略
- [ ] `restart: unless-stopped` 已配置
- [ ] 服务器重启后容器自动启动

---

## 安全加固检查

### 网络安全
- [ ] 生产环境使用 HTTPS（通过反向代理）
- [ ] HTTP 请求重定向到 HTTPS
- [ ] 非必要端口已关闭（防火墙配置）
- [ ] 仅开放 80/443（如有反向代理）或应用端口

### 访问控制
- [ ] 后台路径 `/admin` 不对外公开（或已加固）
- [ ] 如暴露在互联网，考虑添加 IP 白名单

### 数据保护
- [ ] `.env` 文件权限设置为 600（仅所有者可读写）
- [ ] `.env` 文件不在 git 版本控制中
- [ ] 数据库文件权限设置为 600
- [ ] 上传目录权限设置为 755

### 发布前自动化检查

运行 Preflight Check 进行发布前自动化验证：

```bash
# 完整检查（包含构建、测试、运维脚本验证）
npm run preflight

# 快速检查（跳过构建，只验证运行态）
npm run preflight:quick
```

检查项包括：
- [ ] 构建是否成功
- [ ] 健康检查是否通过
- [ ] Smoke Test 是否通过
- [ ] Acceptance Test 是否通过
- [ ] 备份脚本是否正常
